Mencuba beberapa minggu, mempunyai 3 model besar di dalam komputer

Asli Danglive Danglive
3 Mei 2025 16:30 Shanghai
Sejak DeepSeek menyediakan kemungkinan untuk menempatkan LLM secara lokal, saya mulai mencoba menempatkan secara lokal melalui Ollama. Tidak bisa dipungkiri, cara perintah seperti 'ollama run qwen3:14b' sudah sangat sederhana. Ollama langsung mengunduh file model, kemudian bisa langsung masuk ke status percakapan.

Komputer saya dirangkaikan sendiri pada separuh kedua tahun 2021. Pada masa itu, harga kad kawalan grafik sangat tinggi, jadi saya bermula dengan menggunakan kad kawalan grafik 1060. Selepas itu, saya menunggu seketika dan menukarnya kepada 3080 Ti 12GB. Pada masa kini, tidak perlu menukar kad kawalan grafik baru untuk ujian LLM, kerana kad kawalan grafik dengan memori besar masih mahal.

Alhamdulillah, syarikat-syarikat pembuat model besar saling bersaing dengan ketat, menyebabkan model-model yang tidak penuh darah kini semakin mesra dengan pemasangan di atas GPU kelas pengguna. Akhir-akhir ini, saya telah mencuba memasang beberapa model secara tempatan:
qwq:32b-preview-q4_K_M
deepseek-r1:32b
deepseek-r1:14b
gemma3:27b
gemma3:12b
Qwen3:14B

Setelah dicuba, model 32b mempunyai saiz kira-kira 20GB, yang sangat memberatkan untuk GPU 12GB ini, memenuhi sepenuhnya RAM dan kelajuan pengeluaran token terlalu perlahan.

Jadi, parameter 12b dan 14b ini telah dipertahankan, saiz fail model dengan skala parameter ini biasanya kira-kira 9GB. Masing-masing ialah:
deepseek-r1:14b   9.0GB
gemma3:12b         8.1GB
Qwen3:14B            9.3GB

Dari segi kelajuan operasi, yang paling cepat ialah qwen3:14b, penggunaan GPU jauh lebih tinggi, boleh mencapai lebih dari 90%. Selain itu, ia ialah model yang menggabungkan inferens dan bukan inferens, mempunyai ciri khas. Apabila tidak perlu membuat inferens, cukup tambahkan "/no_think" dalam dialog, dan jawapan akan menjadi lebih ringkas dan tepat. Ketika memanggil API, walaupun ia masih akan mengeluarkan sepasang tanda <think> yang kosong, perlu membuat sedikit pembersihan pada string yang dikembalikan, jawapannya sudah cukup mudah untuk diproses seterusnya.

gemma3:12b, saya juga masih menyimpannya kerana ia merupakan model multimodal yang boleh menjelaskan gambar. Hanya sahaja apabila memanggil api, ia agak menyusahkan kerana sentiasa menambahkan sedikit pengenalan dan kesimpulan dalam jawapannya. Untuk pemanggilan program yang hanya memerlukan hasil (seperti menterjemahkan satu segmen perkataan), kita perlu menambah sedikit arahan dan kawalan dalam prompt.

deepseek-r1:14b juga masih disimpan. Karena saya akan memanggil API deepseek di cloud sebagai rujukan lokal, utamanya untuk memeriksa perbezaan antara versi online dan offline secara berkala. Ia mungkin merupakan model yang paling bercakap panjang, tetapi tetaplah cahaya negara, menyimpannya adalah satu perasaan. Apabila r2 keluar, ia mungkin akan digantikan dengan versi tertentu dari r2.
